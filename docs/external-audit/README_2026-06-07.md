## Krátky záver

Toto už nie je iba web pre makléra.

Aktuálny systém je funkčné MVP realitného lead systému:

```text
web / rezervácia
-> Cloudflare Worker API
-> Google Calendar dostupnosť a event
-> Telegram notifikácia
-> Supabase CRM zápis
-> demo dashboard
-> pripravený OpenClaw handoff
```

Projekt je blízko soft production pilotu, ale ešte nie je pripravený na ostré škálovanie ako platený produkt pre ďalších maklérov bez externého auditu.

## Produktová hypotéza

Produkt nemá byť "webstránka pre makléra".

Má to byť broker lead/acquisition suite:

```text
človek chce predať alebo kúpiť nehnuteľnosť
-> príde na osobný web makléra
-> vyplní riadený kvalifikačný flow
-> termín sa overí podľa kalendára
-> maklér dostane kontext a notifikáciu
-> lead sa zapíše do CRM
-> agent/OpenClaw pripraví follow-up
-> dashboard drží prehľad o leadoch, úlohách a obchodoch
```

## Čo je silné

- Jasný vertikálny use case: realitný maklér, nie generický chatbot.
- Custom booking flow bez Calendly, priamo cez Cloudflare Worker + Google Calendar.
- Worker pred vytvorením eventu robí druhý free/busy check.
- Lead sa nestratí iba v kalendári; systém smeruje do Supabase CRM.
- Telegram notifikácie sú súčasťou obchodného flow.
- OpenClaw smer je pripravený: agent, Telegram binding, repo mount, hook koncept.
- SEO/AI discoverability existuje: sitemap, robots, llms.txt, JSON-LD.
- Staging/production workflow je oddelený.
- Projekt má ekonomickú logiku naviazanú na realitnú províziu, nie na cenu one-page webu.

## Čo ešte nie je 100 %

### P0 pred reálnymi klientskymi dátami

- Zamknúť `/dashboard/*` a `/api/dashboard/*`, ak sa má zapnúť reálne CRM čítanie.
- Finálne vyriešiť Jakubov Google Calendar OAuth a kalendár ako source of truth.
- Potvrdiť GDPR/privacy právnym zázemím Jakuba/BOSEN.
- Vyčistiť alebo označiť staging smoke test dáta.
- Nastaviť rozumný `BOOKING_MIN_LEAD_MINUTES`; teraz je `0`.
- Rozhodnúť, či klient dostáva potvrdzovací e-mail.
- Pridať rate limit / Turnstile alebo aspoň spam hardening, ak začne chodiť traffic.

### P1 pred predajom ako produkt

- OpenClaw ešte nemá deterministické Supabase CRM tools.
- Deployed Worker ešte nemá verejný bezpečný OpenClaw hook cez Tunnel/Access.
- Dashboard je zatiaľ prototyp, nie operačný CRM panel.
- Chýba editácia leadov, taskov, poznámok, follow-upov.
- Chýba approval queue pre verejné zmeny webu/listingov.
- Chýba systematická observability/logging vrstva pre Worker.
- CSS/design systém je rozbehnutý, ale veľa štýlov je stále v page-level `<style>` blokoch.

## Pricing hypotéza

Jednoduchý one-page web na slovenskom freelance trhu môže stáť okolo 650 EUR, ale tento systém sa nesmie porovnávať s one-page webom.

Obhájiteľný pricing po audite:

```text
Beta / prvý externý maklér:
2 500 - 3 500 EUR setup
+ 300 - 600 EUR mesačne

Produkčný balík:
5 000 EUR setup
+ 750 - 1 000 EUR mesačne

Prémiový balík:
7 500 - 10 000 EUR setup
+ 1 000 - 1 500 EUR mesačne
+ success fee, ak je tracking a zmluva
```

Pricing 5k setup + 1k mesačne je obhájiteľný iba vtedy, ak systém jasne dodáva:

- web,
- lead funnel,
- AI/agent kvalifikáciu alebo asistenciu,
- calendar/booking,
- CRM zápis,
- notifikácie,
- dashboard,
- monitoring,
- mesačnú optimalizáciu,
- merateľné obchodné výsledky.
