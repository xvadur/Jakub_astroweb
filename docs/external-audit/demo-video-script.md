# Demo video script - 5 až 7 minút

Použi pre Martina aj Martinu. Rozdiel je v závere: Martin = technický/product audit, Martina = UX/content/trust audit.

## 0:00 - 0:30 Kontext

Povedz:

> Som Adam. Pôvodne som zdravotník, nie klasický programátor. Posledný rok denne buildím cez AI, GitHub a Codex. Toto je systém pre konkrétneho realitného makléra Jakuba Olšu. Nie je to iba web. Je to MVP lead systému.

## 0:30 - 1:20 Homepage

Ukáž:

- hero,
- Jakubov príbeh,
- BOSEN zázemie,
- proof/stats,
- služby,
- referenčné predaje,
- CTA na rezerváciu.

Povedz:

> Cieľ nie je pekná stránka. Cieľ je dostať človeka, ktorý zvažuje predaj alebo kúpu nehnuteľnosti, do kvalifikovaného procesu.

## 1:20 - 2:50 Rezervačný wizard

Ukáž:

- zámer,
- typ nehnuteľnosti,
- lokalitu,
- parametre,
- dátum a čas,
- kontakt,
- GDPR checkbox.

Povedz:

> Toto je lead qualification vrstva. Nechcem, aby maklér dostal random formulár. Chcem, aby dostal kontext: kto je človek, čo rieši, akú nehnuteľnosť, kde, kedy a prečo.

## 2:50 - 3:40 API / booking logika

Ukáž len stručne:

- Worker file,
- `/api/availability`,
- `/api/book`,
- Google Calendar mode,
- Supabase,
- Telegram,
- OpenClaw handoff.

Povedz:

> Booking nie je mailto hack. Worker vie overiť dostupnosť, vytvoriť event, zapísať CRM a poslať notifikáciu. Nie všetko je ešte produkčne dotiahnuté, ale jadro existuje.

## 3:40 - 4:30 Dashboard

Ukáž:

- `/dashboard/leady/`,
- demo lead list,
- detail drawer.

Povedz:

> Dashboard je zatiaľ prototyp. Reálne CRM dáta sú na verejnom stagingu vypnuté, kým nebude auth. Toto je jedna z vecí, kde chcem audit.

## 4:30 - 5:20 GitHub / workflow

Ukáž:

- repo,
- branch `staging`,
- docs,
- issues/tickety, ak existujú,
- ako pracuješ s Codexom.

Povedz:

> Toto celé vznikalo rýchlo a iteratívne cez AI-assisted development. Funguje to, ale nechcem si nahovárať, že je to produkčne čisté len preto, že build prejde.

## 5:20 - 6:30 Záver pre Martina

Povedz:

> Od teba by som chcel production/product audit. Čo musí byť fixnuté pred reálnymi leadmi? Čo je len technický dlh? Čo by bol najmenší platený MVP scope? A či je pricing 5k setup + 1k mesačne obhájiteľný, ak to predávam ako realitný lead systém, nie web.

## 5:20 - 6:30 Záver pre Martinu

Povedz:

> Od teba by som chcel UX/content/trust audit. Čo pôsobí amatérsky? Čo znižuje dôveru? Ako má vyzerať homepage a wizard, aby predávajúci človek cítil, že Jakub je bezpečný profesionálny kontakt? A čo mi ako AI-native builderovi najviac chýba vo frontende/dizajne?
